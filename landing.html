Pipeline

Six steps. One command.

The same orchestrator runs in both the CLI and the Streamlit UI. State lives in return dicts — the UI is just a display layer.

1

Schema extraction

Loads CSV, Excel, JSON, or TSV. Infers datetime columns and hidden-numeric columns (e.g. "$1,200" → 1200). Passes only structural metadata downstream — never raw rows.

schema_extractor.py
2

Missing-values analysis

Per-column null counts and percentages, co-missingness correlation between affected columns, high-missing flagging, and a viridis heatmap.

missing_analyzer.py
3

Visualisations

Priority-ranked deterministic charts: correlation heatmap → missing bar → distributions → boxplots → value-counts → scatter plots. Identifier columns (Name, ID, Ticket) are automatically excluded.

plotter.py
4

AI narrative (optional)

Mistral via Ollama reads the actual computed statistics and writes insights as a senior data scientist would — key patterns, data quality issues, correlations, and modelling recommendations.

llm_agents.py
5

Python scripts

Ready-to-run preprocessing code tailored to the specific dataset: drop high-missing columns, impute, one-hot encode, scale, and stratified train/test split.

report_builder.py §5
6

PDF report

fpdf2 assembles a professional, page-capped PDF. If content would exceed 30 pages, lower-priority charts are trimmed first. The most informative content always appears.

report_builder.py
Features

Production-ready, not a toy demo

Every design decision is deliberate and documented in the codebase.

🔒

Sandboxed code execution

LLM-generated plot code runs in an isolated namespace with sanitize_code() guardrails. A 3-attempt retry loop sends errors back to the model for self-correction.

📊

Priority-ranked charts

Charts are returned as (path, caption, priority) tuples. The PDF builder enforces the 30-page cap by dropping lower-priority visuals first.

🧠

Deterministic first

The full pipeline works without any LLM. AI narrative is an additive layer — toggle it on with --ai when Ollama is running.

🩺

Deep missing-values analysis

Co-missingness correlation between affected columns, column type-aware imputation strategies, and copy-paste pandas code per column.

⚡

One orchestrator

pipeline.run_pipeline() is the single entry point. Both the CLI and Streamlit UI call it. No duplicated logic, no diverging behaviour.

🐳

Docker-ready

Two-service Compose setup: eda-pipeline + Ollama sidecar. Data and outputs directories are volume-mounted so files persist across container restarts.

Quick start

Running in under 2 minutes

Fast mode needs no Ollama. AI mode adds 30–90 s and a model download.

Step 1 — Clone and install

# Clone
git clone https://github.com/
  Sujay1709/agentic-eda-pipeline
cd agentic-eda-pipeline

# Create virtual env
python -m venv .venv
source .venv/bin/activate

# Install
pip install -r requirements.txt

Step 2 — Run fast mode (no AI)

# Streamlit web UI
streamlit run app.py
# → open http://localhost:8501

# OR — CLI
python eda_pipeline.py \
  --input data/your_file.csv

# PDF auto-opens on macOS / Linux

Step 3 — Enable AI narrative

# Install Ollama → https://ollama.com
ollama pull mistral

# CLI with AI
python eda_pipeline.py \
  --input data/your_file.csv \
  --ai --model mistral

# Or toggle in the Streamlit sidebar

Step 4 — Generate a test dataset

python3 -c "
import pandas as pd, numpy as np
df = pd.DataFrame({
  'price': np.random.uniform(100,500,200),
  'volume': np.random.randint(1000,50000,200),
  'category': np.random.choice(
    ['Tech','Finance','Health'],200)
})
df.to_csv('data/sample.csv',index=False)
"
